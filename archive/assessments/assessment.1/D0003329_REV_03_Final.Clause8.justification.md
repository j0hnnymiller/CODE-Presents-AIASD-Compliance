## Partial Compliance Gaps

### Gap #1: Explicit Identification Scheme (§8.1.1)

#### IEC 62304 Requirement

> §8.1.1: "establish a scheme for the unique identification of CONFIGURATION ITEMS and their VERSIONS"

#### D0003329 Content

§6.12 Para 2: "Categories (or lists) of items that will be controlled"
§6.12 Para 5: "unique name and version number"

#### Gap Determination - **PARTIAL**

**How Determined**:

1. IEC 62304 requires: "**scheme** for unique identification"
2. D0003329 mentions: "categories of items" and "unique name and version"
3. Gap: Requirement split across paragraphs; "scheme" not explicitly stated

**Why Partial (Not Non-Compliant)**:

- Unique identification IS addressed (Para 5)
- Version identification IS addressed (Para 5)
- However, explicit "scheme" requirement could be more prominent

**Evidence**:

- Coverage exists but fragmented across §6.12
- Could consolidate into clear §8.1.1 statement

**Impact**: Minor - teams will develop identification schemes, but explicit guidance would improve consistency

### Gap #2: SOUP Identification Elements (§8.1.2)

#### IEC 62304 Requirement

> §8.1.2: Document for each SOUP: (a) title, (b) MANUFACTURER, (c) unique SOUP designator

#### D0003329 Content

§6.12 Para 2: "how SOUP is handled"

#### Gap Determination - **PARTIAL**

**How Determined**:

1. IEC 62304 specifies THREE elements: title, manufacturer, designator
2. D0003329 states "how SOUP is handled" (generic)
3. Gap: Three specific elements not enumerated

**Why Partial (Not Non-Compliant)**:

- SOUP IS mentioned in configuration management
- However, specific documentation elements (a, b, c) not explicit

**Evidence**:

- §6.4 (Architecture) addresses SOUP description
- §6.12 addresses SOUP handling
- Missing: Explicit three-element documentation requirement in §6.12

**Significant Gap** within partial compliance - SOUP identification is critical for reproducibility

**Recommended Enhancement**:

```
SOUP Configuration Items shall be documented including:
a) Title (name of SOUP component)
b) Manufacturer/supplier
c) Unique designator (version, release date, patch number)
```
