## 6. RECOMMENDATIONS

### 6.1 Immediate Actions (Critical Priority)

#### R1: Add Software Archival Requirements (Addresses G1)

**Section**: §6.9
**Action**: Add new subsection to Software Release section:

```markdown
### Software Archival (All Classes)

The released medical device software, all configuration items, and associated
documentation shall be archived for a retention period equal to the longer of:

- The lifetime of the medical device software as defined by the manufacturer, or
- The period specified by applicable regulatory requirements (e.g., FDA 21 CFR 820.180)

Archival procedures, including storage location, media, access controls, and retrieval
processes, shall be documented in the Software Configuration Management Plan (D0004170).

For Class B and C devices, archival records shall include:

- Released software executables and source code
- Build environment specifications and tools
- All verification and validation documentation
- Software development plan and associated deliverables
- Known residual anomalies and risk assessments
```

**Implementation**: Update D0004170 (CM Plan template) to include archival procedures section.

#### R2: Expand SOUP Specification Guidance (Addresses G2)

**Section**: §6.4
**Action**: Enhance SOUP requirements:

```markdown
### SOUP Specification Requirements (Class B/C)

When SOUP items are identified within the software architecture, the following shall
be documented:

Functional and Performance Specifications (§5.3.3):

- Required functionality necessary for intended use in the medical device
- Performance requirements (response time, throughput, resource consumption)
- Limitations or constraints relevant to the medical device application

System Requirements for SOUP Operation (§5.3.4):

- Processor type, speed, and architecture requirements
- Memory requirements (RAM, storage)
- Operating system type, version, and configuration
- Required libraries, frameworks, and dependencies
- Communication protocols and network requirements
- Display and user interface requirements (if applicable)

SOUP items and their specifications shall be documented in the Software Architecture
(D0004197) and tracked in the Software Configuration Management Plan (D0004170) with
version control requirements.
```

**Implementation**: Update D0004197 template to include SOUP specification checklist.

### Software Delivery Procedures (All Classes)

Procedures shall be established to ensure reliable delivery of released medical device
software to the point of use without corruption or unauthorized change. These procedures

- Media labeling standards and requirements
- Packaging methods ensuring software protection
- Protection against unauthorized access or modification
- Storage conditions (temperature, humidity, security)

Delivery procedures shall be documented in or referenced by the Software Development
